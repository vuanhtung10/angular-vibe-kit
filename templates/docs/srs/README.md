# SRS (source requirements)

This folder holds the **Markdown sources** that `/plan` reads. Upstream SRS/spec docs arrive in
mixed formats — PDF, Word `.docx`, PowerPoint, Excel, screenshots with text, audio, YouTube
transcripts, EPub, ZIP. Convert them to Markdown first using **Microsoft Markitdown** (better than
`pandoc` for PDF/PPTX/XLSX/OCR; LLM-token-optimized output).

Recommended path:
1. `markitdown` must be installed once on your dev machine (the Angular project itself stays pure
   Node — do NOT add it to `devDependencies`):
   ```bash
   # Windows (recommended — uv handles Python + venv for you)
   winget install astral-sh.uv
   uv tool install "markitdown[all]"

   # macOS / Linux
   uv tool install "markitdown[all]"
   # or:  pip install 'markitdown[all]'
   ```
2. Convert (or use the kit's `/convert-srs` slash command — it wraps everything below):
   ```bash
   markitdown path/to/spec.docx -o docs/srs/<feature>.md
   # multi-module PDFs/PPTs: run /convert-srs, it auto-splits and indexes
   ```

The command handles: split by H1 into one file per module, extract images under `_images/`,
and prepend a row to this README's index below.

## Convention

```
docs/srs/
├── README.md                      ← this file (index + image dir pointer)
├── module-5-order-management.md   ← generated by /convert-srs or `markitdown ... -o`
├── module-6-invoicing.md
└── _images/                       ← extracted by Markitdown; paths are relative to each .md
```

## Usage

This folder is **not** auto-read — `/plan` doesn't scan it on its own. Point to the file
explicitly when you start planning:

```
/plan

Đọc yêu cầu từ docs/srs/module-5-order-management.md
```

Keep these files committed — they're the record of what was actually asked for each module,
useful when a later module needs to know how an earlier one was scoped.
